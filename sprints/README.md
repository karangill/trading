# Sprints Index

Each Sprint is a 3-6 week time-boxed framework-iteration block within a Quest. Sprint files follow the Plan → In-Progress Tracking → Review → Handoff lifecycle.

Sprints are distinct from Quests. Sprints are time-boxed (3-6 weeks) and exist to iterate the framework. Quests are outcome-defined (financial milestones) and contain one or more Sprints. See `Trading_OS.md` § 3.

## File naming convention

`sprint_N_short-name.md`

## Lifecycle status values

- **Planned** — file exists, Plan section filled, Sprint hasn't started
- **Active** — Sprint in progress, Tracking section being populated
- **Completed** — Review section filled, Handoff to next Sprint done
- **Extended** — Sprint window formally extended (insufficient data, framework needs more iteration time)
