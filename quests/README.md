# Quests Index

Each Quest is an outcome-defined financial milestone arc. Quest files follow the Plan → In-Progress Tracking → Review → Handoff lifecycle.

Quests are distinct from Sprints. Quests are outcome-defined (financial milestones). Sprints are time-boxed framework iteration blocks within Quests. See `Trading_OS.md` § 3.

## File naming convention

`quest_N_short-name.md`

## Lifecycle status values

- **Planned** — file exists, Plan section filled, Quest hasn't started
- **Active** — at least one Sprint has begun within this Quest
- **Completed** — Quest success criteria met, Review filled, Handoff to next Quest done
- **Restructured** — Quest plan was formally changed mid-Quest
